# DocNexus Outreach Platform — Consolidated Session Context

We are building a healthcare outreach CRM MVP called “DocNexus Outreach Platform” using the engineering blueprint already provided.

## Stack

* Next.js 14 App Router
* TypeScript strict mode
* Tailwind CSS
* shadcn/ui (slate base color)
* Prisma + SQLite
* Zod validation
* React Hook Form
* Recharts
* Jest + ts-jest
* Groq AI integration later

---

# Engineering Constraints

* Enterprise SaaS UI (Salesforce/HubSpot style)
* Keep components under ~150 LOC where possible
* Shared Zod schemas in src/lib/validations.ts
* All API routes return JSON only
* Use Prisma transactions where appropriate
* Filters must sync to URL params
* No JS-side filtering after fetching
* SQLite DB must persist at prisma/dev.db
* No Redux or React Query
* Prefer server components unless interactivity required
* Never modify unrelated files
* Make realistic commits phase-by-phase
* Keep architecture clean and explainable

# Important Fixes Already Applied

## Next.js Config

next.config.ts was invalid for current Next.js version.

Fixed by:

* renaming to next.config.mjs
* updating tsconfig includes accordingly

---

## Prisma SQLite Path

DATABASE_URL originally used:

file:./prisma/dev.db

This created:
prisma/prisma/dev.db

Corrected to:

file:./dev.db

because Prisma resolves SQLite paths relative to schema.prisma.

Actual DB path now:
prisma/dev.db

---

## Jest + Next.js TS Conflict

Next.js picked up *.test.ts files causing TS errors during dev/build.

Fixed by:

* adding tsconfig.jest.json
* excluding **/*.test.ts from main tsconfig
* keeping Jest isolated from Next.js compiler

---

## Windows Prisma DLL Lock Issue

On Windows, stale Node processes locked Prisma DLLs causing:

* hanging builds
* localhost not loading
* stuck next dev

Fix:

* kill stale node processes
* rerun prisma generate
* restart dev server
